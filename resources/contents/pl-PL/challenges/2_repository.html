Tworzenie nowego repozytorium na Twoim komputerze.

Repozytoria

Repozytorium jest zbiorem powiązanych elementów. W naszym przypadku, kiedy piszemy aplikację jest to kolekcja plików związanych z danym projektem. Wyobraź sobie to jako folder z projektem zawierający wszystkie powiązane z nim pliki. W rzeczywistości tak właśnie będzie to wyglądało. Repozytoria często nazywa się "repo".

Wskazujesz Gitowi czym jest Twój projekt i zaczyna on śledzić wszystkie zmiany w wybranym folderze. To czyni go repozytorium Git: folder z elementami śledzony przez Git. Git śledzi jakie elementy są dodawane, usuwane, a nawet kosmetyczne zmiany jak korekta jednej litery w pojedynczym pliku. Wszystko to plus kto dokonał zmiany i kiedy są śledzone przez Git. W programowaniu, śledzenie takich zmian nazywane jest kontrolą wersji.

Korzystanie z terminala

Terminal (lub Bash) jest sposobem na poinstruowanie Twojego komputera co ma zrobić poprzez wpisywanie poleceń zamiast interakcji z aplikacją poprzez interfejs graficzny. Możliwe jest zmienianie nazw plików, otwieranie ich, tworzenie nowych katalogów i poruszanie się między nimi, a także wiele więcej za pomocą samych komend. Możesz nawet korzystać z edytora tekstowego by tworzyć program (jak Vim) w swoim terminalu i nigdy nie musieć go zamykać!

Oprócz nawigacji po swoim komputerze możesz także uruchamiać programy korzystające z wiersza poleceń (ang. CLI - Command Line Interface), co oznacza iż mogą one być uruchamiane za pomocą poleceń wydawanych w terminalu. Git jest jednym z nich. Pierwsza część polecenia mówi Twojemu komputerowi, że rozmawiasz z Gitem. Następne składowe polecenia są poleceniami i różnymi opcjami mówiącymi Gitowi jak ma się zachować.

W Git-it nauczysz się paru podstawowych akcji wiersza poleceń (jako dodatek do poleceń Git) do nawigowania po Twoim komputerze; są opisane w poszczególnych krokach.

Tworzenie repozytorium

Utworzysz nowy folder projektu i zainicjujesz w nim nowe repozytorium Git. Na potrzeby tego wyzwania nazwiemy ten folder 'hello-world', ale możesz wybrać inną nazwę jeśli posiadasz już folder o tej nazwie (jest to popularna nazwa dla pierwszych projektów).

Czas otworzyć terminal!

W oknie Twojego terminala wprowadź następujące polecenia, jedno po drugim, zatwierdzając przyciskiem enter/return po każdej wprowadzonej linii.

Najpierw, utwórz nowy folder:
Rada: mkdir jest skróceniem angielskiego wyrażenia make directory (pol. utwórz katalog)

mkdir hello-world

Następnie przejdź do tego katalogu:
Rada: cd jest skróceniem angielskiego wyrażenia change directory (pol. zmień katalog)

cd hello-world

Na koniec, powiedz Gitowi by zainicjalizował (zaczął śledzić zmiany) folder, w którym się znajdujesz:

git init

Ostatnie polecenie powinno zwrócić informację zawierającą "Initialized empty Git repository". Inne polecenia nie powinny zwrócić nic.

Udało Ci się! Jeśli chcesz się upewnić, że inicjalizacja repozytorium Git przebiegła poprawnie, wpisz git status i jeśli wykonanie tego polecenia nie zwróci 'fatal: Not a git repository...', masz to jak w banku!
