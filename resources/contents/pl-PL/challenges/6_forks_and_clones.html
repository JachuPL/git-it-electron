Forkowanie projektów z GitHub.com i klonowanie ich lokalnie.

Forki

Jak dotąd utworzyłeś projekt lokalnie i wypchnąłeś do repozytorium na GitHub.com, ale to tylko połowa zabawy! Druga połowa to współpraca z innymi ludźmi nad innymi projektami.

Kiedy forkujesz repozytorium, tworzysz jego kopię na swoim koncie GitHub. Twoja sforkowana kopia zaczyna życie jako repozytorium zdalne - istnieje po prostu na Twoim koncie GitHub, nie na Twoim komputerze. Forki są używane do tworzenia własnych wersji projektu (ta dywersja od oryginału jest jak wybranie drugiej drogi na skrzyżowaniu) lub współpracy nad nim (jak łatanie błędów czy nowe funkcjonalności).

By pobrać sforkowane repozytorium ze swojego konta GitHub na swój komputer należy je sklonować. Klonowanie powoduje skopiowanie repozytorium zdalnego na Twój komputer w celu umożliwienia pracowania lokalnie.

Forkowanie repozytorium Patchwork

Projekt nad którym pochylimy się tym razem znajduje się pod adresem github.com/jlord/patchwork. Przejdź na tę stronę i kliknij przycisk 'Fork' w górnej prawej części widoku. Kiedy animacja forkowania się zakończy, będziesz posiadać kopię repozytorium na swoim koncie. Skopiuj adres HTTP z paska adresu swojej przeglądarki, to jest adres Twojego forka na serwerach GitHuba.

Klonowanie forka lokalnie

Teraz za pomoca terminala sklonujesz repozytorium na swój komputer. Nowy folder zostanie automatycznie utworzony by przechować repozytorium, więc nie ma potrzeby tworzyć go samodzielnie. Upewnij się jednak że nie klonujesz go do folderu w innym repozytorium!

Upewnij się zatem, że nie jesteś w folderze 'hello-world' z poprzednich wyzwań. By wycofać się z niego w celu sklonowania forka, podążaj za poniższymi krokami:

Back out of your 'hello-world' folder:
Tip: the two dots mean step out of a directory one level

cd ..

Now that you're no longer in another Git repository, clone your fork:

git clone <URLFROMGITHUB>

Go into the folder it created for your local copy of the fork (in this case, named 'patchwork').

cd patchwork

Now you've got a copy of the repository on your computer and it is automatically connected to the remote repository (your forked copy) on your GitHub account. Type git remote -v to see that the address to the fork is already set up.

Connect to the Original Repository

What if the original repository you forked happens to change? You'll want to be able to pull in those changes too. So let's add another remote connection, this time to the original, github.com/jlord/patchwork, repository with its URL.

You can name this remote connection anything you want, but typically people use the name 'upstream'; let's use that for this.

git remote add upstream https://github.com/jlord/patchwork.git

To be sure you have the correct remotes set up, type git remote -v to list out the addresses you have stored. You should have an 'origin' remote with your fork's address and then an 'upstream' remote with the address to the original, the URL noted above in this step.
